Процедура генерации ключей для шифрования, зашифровки и расшифровки файлов
1. Генерация ключей:
gpg --full-generate-key
Отвечаем на все вопросы (выбираем алгоритм, пригодный для шифрования, указываем срок действия ключа, длину, свои данные и парольную фразу)

2. Просмотр ключей, установленных в системе:
gpg --list-key
/home/tmp/.gnupg/pubring.kbx
----------------------------
pub   rsa3072 2024-11-09 [SC] [годен до: 2025-02-07]
      58BA97576F10BF4905F024D169BE97FB30E64E45
uid         [  абсолютно ] Vladislav Pobedinskii (student) <skyzoo.van@gmail.com>
sub   rsa3072 2024-11-09 [E] [годен до: 2025-02-07]

3. Эскпортируем открытый ключ в файл:
gpg --export skyzoo.van@gmail.com > <имя файла>

4. Отправляем полученный файл с открытым ключом субъекту, который должен прислать зашифрованное сообщение.
Субъект шифрует свой файл при помощи нашего открытого ключа.

5. Импорт открытого ключа в систему субъекта
gpg --import <имя файла>

Просмотр: gpg --list-key

6. Процесс шифрования файла открытым ключом:
gpg --out <имя файла> --recipient skyzoo.van@gmail.com --encrypt <имя входного файла>

Субъект отправляет зашифрованный файл нам для дешифровки

7. Процесс расшифровки файла закрытым ключом:
gpg --out <имя выходного файла> --decrypt <имя зашифрованного файла>

Вводим парольную фразу, сопоставленную секретному ключу
Просматриваем расшифрованный файл
